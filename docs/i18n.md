# Internationalization

NotifyHub starts with English and French. English is the source and fallback locale.

## Resources

Translations live in:

```text
packages/i18n/src/locales/en.ts
packages/i18n/src/locales/fr.ts
```

Both files must expose identical translation keys. Tests verify key parity.

## Locale resolution

Discord locales such as `en-US`, `en-GB`, and `fr` are normalized to a supported NotifyHub locale. Unsupported locales fall back to English.

## Contributor rules

- Add English and French translations in the same pull request.
- Keep keys descriptive and stable.
- Do not concatenate translated fragments when a complete sentence can be translated.
- Use interpolation for variable values.
- Keep code, comments, commit messages, and documentation in English.
- Never use provider-generated text as a translation key.

## Adding a language

1. Add the locale resource.
2. Add it to `SUPPORTED_LOCALES`.
3. Update locale resolution and Discord command localizations.
4. Add parity and fallback tests.
5. Update the README language table.
